# 卜筮正宗十八论与十八问答读书笔记

本笔记把《卜筮正宗》的核心章节转成可执行的学习任务。它不复制古籍全文，而是把每个主题拆成：解决的问题、断法焦点、案例字段、练习任务和可连接的规则卡。

## 1. 使用方法

读这份笔记时不要直接背结论。更有效的流程是：

1. 先读对应主题，明确它解决哪一种断卦问题。
2. 再打开 `data/classic_notes.json`，查看字段和练习任务。
3. 对照 `data/rules.json`，确认是否已有规则卡。
4. 找案例时用 `data/case_schema.json` 留痕。
5. 只有带反馈的案例才能提高规则置信度。

## 3. 十八问答笔记

十八问答适合转成案例训练集。每一问都应至少收集 5 到 10 条带反馈案例。

| 次序 | 问答主题 | 案例化方向 |
|---|---|---|
| 1 | 三传克用 | 拆分月克、日克、动克，记录是否有原神救应 |
| 2 | 回头克 | 记录动爻化出之爻反克本爻，标明被克者身份 |
| 3 | 生用神者为原神 | 判断帮助是否有力，是否能到用神 |
| 4 | 三合成局 | 记录成局五行，对用神是助是害 |
| 5 | 反吟、伏吟 | 区分翻转往复和迟滞内耗 |
| 6 | 旬空 | 记录出空、填实、逢冲、动化实条件 |
| 7 | 月破 | 区分用神月破、原神月破、忌神月破 |
| 8 | 用神不现 | 查伏神、飞神、月日代用和出伏条件 |
| 9 | 进退神 | 记录趋势增强或减弱是否被反馈验证 |
| 10 | 冲合互见 | 标注先合后冲、先冲后合，以及应期触发点 |
| 11 | 四生墓绝 | 把墓、绝、长生等写成阶段信息 |
| 12 | 六冲六合 | 不直接断吉凶，按题型和用神关系解释 |
| 13 | 三刑六害 | 只在涉及用神、世应、动爻时重点记录 |
| 14 | 独静独发 | 独发为事件触发点，独静为众动中的稳定点 |
| 15 | 尽静尽发 | 全静重月日，全动重动变传导 |
| 16 | 用神多现 | 比较旺衰、动静、贴题和世应距离 |
| 17 | 规则冲突 | 保留互相竞争的解释，等待反馈裁决 |
| 18 | 卦验与不验 | 建立命中、偏差、失验、不可验证标签 |

## 4. 规则抽取规范

每次从古籍或案例抽规则，必须写清楚：

- `source_refs`：来源，不可只写“古书说”。
- `linked_rule_ids`：如果已有规则卡，必须连接。
- `extraction_fields`：案例录入时要收集哪些证据。
- `practice_tasks`：下一步如何验证。
- `conflict_note`：与其他规则冲突时保留争议。

## 5. 本轮外部检索记录

2026-06-20 本轮只把可复查的公开网页、古籍页面和课程目录作为资料线索。部分古籍页面读取受限，因此没有把新抓取文本当作本轮证据。B站 `bili search "六爻 卜筮正宗 十八论"` 可返回课程线索，但只记录为 C 类课程线索，不升级为规则。

## 6. 下一步细化任务

1. 给十八论每一论补至少 3 条古籍原章节定位。
2. 给十八问答每一问补至少 5 条反馈案例。
3. 在 `data/rules.json` 中继续补齐缺失规则卡，例如回头克、三合成局、独静独发。
4. 增加案例库后，按 `hit_or_miss` 统计每条规则的可靠性。
